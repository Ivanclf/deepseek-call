# Deepseek 调用页面设计文档

## 目标
使用 Node.js 后端与原生前端实现一个 Deepseek 调用页面，满足历史对话管理、默认深度思考、以及流式返回展示。

## 设计任务列表

### 1. 项目结构搭建
- [x] 创建 `history/` 文件夹用于存储对话历史 JSON 文件
- [x] 创建 Node.js 项目基础文件：`package.json`、`server.js`
- [x] 创建前端页面文件：`index.html`、`style.css`、`app.js`
- [x] 确定后端静态资源服务和 API 路由结构

### 2. 后端设计（Node.js）
- [x] 提供静态文件服务，渲染前端页面
- [x] 实现对话历史文件读取接口：根据 `conversationId` 返回对应 JSON
- [x] 实现对话历史保存接口：写入/追加消息到 `history/{conversationId}.json`
- [x] 设计 Deepseek 请求封装逻辑
  - [x] 组装请求体：包含 `conversationId`、`messages`、`enableDeepThinking: true`
  - [x] 支持本地模拟 Deepseek 响应流程（不联网）
- [x] 设计流式返回接口
  - [x] 使用 Server-Sent Events (SSE) 或分块传输模拟流式返回
  - [x] 将模拟的 Deepseek 输出分段推送给前端
- [x] 规范 JSON 历史结构：`conversationId`、`createdAt`、`messages`数组

### 3. 前端设计（原生 JS）
- [x] 页面元素布局
  - [x] 输入框：用户问题输入
  - [x] 发送按钮：触发会话请求
  - [x] 对话列表区：展示全部历史消息
  - [x] 流式输出区：展示 Deepseek 增量返回内容
  - [x] 对话选择/新建控件：切换或创建对话
- [x] 对话历史读取与渲染
  - [x] 加载当前对话历史 JSON
  - [x] 渲染每条消息区分 `user` 与 `assistant`
- [x] 请求发送逻辑
  - [x] 点击发送按钮时读取当前输入
  - [x] 先写入用户消息到本地历史文件
  - [x] 调用后端 Deepseek 接口，携带完整历史消息
- [x] 流式返回处理
  - [x] 接收流式事件或分段响应
  - [x] 实时追加渲染返回内容到输出区
  - [x] 更新请求状态显示：`正在请求`、`接收中`、`完成`
  - [x] 同时展示 Deepseek 的思考过程文本
- [x] 本地模拟 Deepseek 结果展示
  - [x] 即便无网络，也可以使用本地预设响应段落展示流式效果

### 4. 对话历史管理
- [x] 设计对话 JSON 文件命名规范，例如 `conversation-{timestamp}.json`
- [x] 新建对话时生成初始 JSON 模板
- [x] 每次请求前读取当前对话完整历史并打包进 Deepseek 请求体
- [x] 请求完成后将 assistant 回复追加到历史文件
- [x] 页面刷新后仍能恢复当前对话内容

### 5. 深度思考默认开启
- [x] 在后端请求封装中加入 `enableDeepThinking: true`
- [x] 前端请求体中不提供关闭选项，默认启用深度思考
- [x] 设计文档说明该功能为默认行为，无需额外配置

### 6. 流式返回效果设计
- [x] 后端模拟多段响应，逐步推送文本片段
- [x] 前端接收并逐段展示，形成“边接收边渲染”体验
- [x] 提供请求进度或状态指示
- [x] 在设计文档中强调即使离线场景也支持流式返回感知

### 7. 扩展与维护
- [x] 预留接口替换点：未来可轻松接入真实 Deepseek API
- [x] 保持历史数据结构兼容真实接口
- [x] 文档说明无需联网，但后端保留“请求与响应抽象层”
- [x] 整体设计以简单可扩展、易维护为原则

## 关键设计说明
- 后端使用 Node.js 提供静态页面和 API，前端使用纯 HTML/CSS/JS 实现交互
- 所有对话历史以 JSON 文件持久化，保存在 `history/` 下，每个对话单独存档
- 每次发送请求时，后端将当前对话历史全部包装到 Deepseek 请求结构中
- 默认启用深度思考功能，无需额外用户配置
- 返回结果采用流式渲染，页面逐步显示 Deepseek 回复内容
